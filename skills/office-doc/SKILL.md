## 何时使用

要写报告、方案、立项说明、工作汇报、制度草案、公告、说明书。

## 常用结构

**汇报 / 报告**
1. 背景与目标
2. 现状与问题
3. 方案 / 建议（分点）
4. 资源与时间
5. 风险与对策
6. 结论与请示事项

**方案 / 提案**
1. 痛点
2. 目标与范围（含不做什么）
3. 方案对比（可选表格）
4. 实施步骤与里程碑
5. 预算 / 人力粗算（有数据才写）

**通知 / 公告**
1. 事由
2. 具体安排（时间地点对象）
3. 注意事项
4. 联系人

## 格式

- 标题层级：`#` `##` `###`，避免跳级
- 数据用表格；步骤用编号列表
- 结尾列「附件清单」如有

## 禁止

- 不虚构政策条文、审批结果
- 用户要 Word 时：先给结构化 Markdown，并说明可用 pandoc/手动粘贴转换
