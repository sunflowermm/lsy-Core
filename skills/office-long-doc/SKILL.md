## 何时使用

标书、可行性报告、白皮书、制度文件、>3000 字需分阶段交付的文稿。

## 流程（四阶段）

### 1. 骨架（必做）

- 读者、目的、篇幅、禁忌
- 章节目录 + 每节 1 句要点
- **等用户确认目录后再写正文**

### 2. 分节扩写

- 每次只写 **1–2 节**，写入工作区 `docs/<项目名>/ch-NN.md`
- 节末附：本节字数、待补材料 `[TODO]`

### 3. 一致性 Pass

- 术语表（同一概念一词）
- 数字、日期、人名跨章一致
- 交叉引用「见第 X 节」是否正确

### 4. 合并交付

- `docs/<项目名>/full.md` 或 docx（office-docx）
- 目录页 + 修订摘要

## 状态文件

在工作区维护 `docs/<项目名>/STATUS.md`：

```markdown
# 进度
- [x] 大纲确认 2026-06-09
- [ ] 第3章
- [ ] 全文合并
开放问题：…
```

## 禁止

- 未确认大纲就写全文
- 单条消息塞 5000+ 字（应分文件）
